# 实验一：HTML 静态网页设计实现计划

## 一、任务目标

在当前项目中完成“实验一：HTML 静态网页设计”。

本实验只实现前端静态系统原型，要求综合使用 Vue + ElementUI 完成一组可点击、可跳转、可展示、具备响应式布局的静态页面。

不要实现后端接口、SpringBoot、数据库、MyBatis、真实登录认证、真实数据持久化。实验一中的用户列表数据使用前端静态模拟数据即可。

## 二、技术边界

优先沿用当前项目已有技术栈。

如果当前项目已经是 Vue 项目，则在现有 Vue 项目上修改和补充页面、路由、组件、样式文件。

如果当前项目尚未搭建完整结构，则按 Vue + ElementUI 的静态前端项目组织代码。

不要把实验二、实验三的要求提前混入实验一：

* 不需要 SpringBoot Controller。
* 不需要 Service。
* 不需要 Dao / Mapper。
* 不需要数据库表。
* 不需要 Axios 请求。
* 不需要从服务器读取数据。
* 不需要真实注册、真实登录、真实修改密码。

## 三、需要完成的页面

至少完成以下页面：

1. 登录页面
2. 注册页面
3. 忘记密码 / 修改密码页面
4. 主页
5. 信息页面
6. 用户列表页面

页面之间要形成一个完整静态系统。按钮和链接不能只是摆设，需要能触发页面跳转、表单校验、主题切换、列表分页等前端行为。

推荐路由结构：

```text
/login
/register
/forgot-password
/home
/info
/users
```

如果项目没有使用 vue-router，则用当前项目中最合适的方式实现页面切换，但必须保证验收时能完整演示页面流程。

## 四、页面功能与交互要求

### 1. 登录页面

页面内容：

* 用户名输入框
* 密码输入框
* 登录按钮
* 注册用户入口
* 忘记密码入口
* 第三方登录展示区，如 QQ 登录、微信登录、支付宝登录，可以作为静态按钮展示

交互要求：

* 用户名不能为空。
* 密码不能为空。
* 点击“登录”后，如果校验通过，跳转到主页。
* 点击“注册用户”后跳转到注册页面。
* 点击“忘记密码”后跳转到忘记密码 / 修改密码页面。
* 第三方登录按钮可以弹出提示，例如“静态原型暂未接入第三方登录”。

响应式要求：

* 以 800px 作为主要断点。
* 大屏时可采用左右分栏或横向布局。
* 小屏时改为单列布局，表单居中，宽度适配手机屏幕。
* 注意实验文档中图注出现“小于 650px”的不一致表述，但正文优先采用 800px 断点。代码和报告中按 800px 解释。

### 2. 注册页面

页面内容：

* 用户名
* 密码
* 邮箱
* 出生年月
* 可选头像上传展示区或静态占位区
* 注册按钮
* 返回登录按钮

表单验证：

* 用户名不能为空。
* 密码不能为空。
* 邮箱必须符合 email 格式。
* 出生年月必须填写。
* 校验通过后弹出注册成功提示，并跳转回登录页或主页。

响应式要求：

* 屏幕宽度 > 800px 时使用两列展示，例如左侧表单、右侧头像上传或展示区。
* 屏幕宽度 ≤ 800px 时使用单列布局。

### 3. 忘记密码 / 修改密码页面

页面内容：

* 用户名
* 邮箱或验证信息
* 新密码
* 确认密码
* 确认修改按钮
* 返回登录按钮

表单验证：

* 用户名不能为空。
* 新密码不能为空。
* 确认密码不能为空。
* 新密码与确认密码必须一致。
* 校验通过后弹出修改成功提示，并跳转回登录页。

响应式要求：

* 屏幕宽度 > 800px 时使用两列展示。
* 屏幕宽度 ≤ 800px 时使用单列布局。

### 4. 主页

主页需要体现实验文档中的大屏和小屏差异。

大屏布局应包含：

* 顶部区域
* 左侧菜单
* 欢迎信息，例如“欢迎 张三”
* 图片区域
* 主体内容区域，例如日历、信息卡片、公告栏等

小屏布局：

* 保留主要内容区域。
* 左侧菜单可以隐藏、折叠或改为顶部菜单。
* 页面不能横向溢出。
* 保证手机宽度下仍然可读、可点击。

断点处理：

实验文档正文写的是 800px，图注写的是 1000px，存在不一致。实现时采用一个明确断点。建议使用 1000px 控制“是否显示左侧菜单”，使用 800px 控制表单类页面的单双列变化，并在报告中说明：

* 表单页面以 800px 为断点。
* 主页菜单展示以 1000px 为断点，使展示效果更接近图注。

### 5. 主题 / 背景风格切换

必须实现主题切换功能。

在主页左侧菜单或顶部菜单中增加主题切换入口：

* 蓝色风格
* 黄色风格

点击“蓝色风格”后，页面主题切换为蓝色系。

点击“黄色风格”后，页面主题切换为黄色系。

主题切换至少影响：

* 顶部栏背景色
* 左侧菜单背景色
* 当前选中菜单项
* 卡片边框或背景
* 按钮主色调

实现方式建议：

* 使用根容器 class，例如 `theme-blue`、`theme-yellow`。
* 点击菜单时修改当前主题变量或 class。
* 不要在大量元素上写重复行内样式。

### 6. 信息页面

信息页面用于补全“完整系统”的页面结构，可以设计为个人信息页、系统说明页或信息展示页。

页面应包含：

* 页面标题
* 信息卡片
* 基本信息展示
* 可点击返回主页或切换页面的按钮

数据使用静态模拟数据。

### 7. 用户列表页面

点击“用户管理”菜单后进入用户列表页面。

必须使用 ElementUI 的表格组件：

* 使用 `el-table`
* 使用 `el-table-column`
* 使用分页组件，例如 `el-pagination`

表格字段建议：

* 编号
* 用户名
* 性别
* 年龄
* 邮箱
* 地址
* 操作

数据要求：

* 使用前端静态数组模拟。
* 不要请求后端。
* 不要连接数据库。

分页要求：

* 至少能展示当前页数据。
* 支持切换页码。
* 支持每页条数设置更好，但不是必须。
* 操作列可以放“查看”“编辑”“删除”按钮，点击后用消息提示模拟操作即可。

## 六、代码风格要求

### 1. HTML / Vue 模板

模板结构要清楚。

不要写大量无意义嵌套：

```html
<div>
  <div>
    <div>
      ...
    </div>
  </div>
</div>
```

应按页面区域组织：

```html
<div class="login-page">
  <section class="login-card">
    ...
  </section>
</div>
```

表单项要尽量完整：

* label 清楚
* placeholder 清楚
* 表单校验规则清楚
* 按钮含义明确

### 2. CSS

CSS 要集中管理，避免大量行内样式。

允许使用：

* 全局 CSS
* 页面 scoped style
* 主题 CSS
* 响应式 CSS

不推荐大量使用：

```html
<div style="width: 100px; color: red;">
```

全局建议加入：

```css
* {
  box-sizing: border-box;
}
```

布局优先使用 flex 或 grid。

不要使用混乱的 float 布局。若已有 float 布局，必须处理清除浮动和父元素高度塌陷问题。

CSS 语法必须完整：

```css
.card {
  padding: 20px;
  border-radius: 8px;
}
```

不要漏冒号、分号、大括号。

### 3. JavaScript

变量和函数命名要有明确含义。

推荐：

```js
const loginForm = ...
const userList = ...
const currentTheme = ...
function handleLogin() { ... }
function validatePasswordConfirm() { ... }
```

避免：

```js
const a = ...
const data1 = ...
function test() { ... }
```

相同逻辑要封装，不要复制粘贴。

表单校验、主题切换、分页计算、菜单跳转等逻辑应分别写清楚。

### 4. Vue / ElementUI

优先使用数据驱动，不要频繁直接操作 DOM。

推荐：

```html
<el-table :data="pagedUserList">
```

不要在 Vue 页面中大量写：

```js
document.getElementById(...).innerHTML = ...
```

ElementUI 组件应规范使用：

* `el-form`
* `el-form-item`
* `el-input`
* `el-button`
* `el-table`
* `el-pagination`
* `el-menu`
* `el-card`

### 5. 静态数据

用户列表、信息页面、欢迎用户名等数据全部使用前端静态数据。

示例：

```js
const users = [
  {
    id: 1,
    name: '张三',
    gender: '男',
    age: 20,
    email: 'zhangsan@example.com',
    address: '湖北武汉'
  }
]
```

不要把静态数据散落在模板中。应放在 data、computed、ref 或单独数据文件中。

## 七、响应式布局详细要求

### 表单页面

适用于：

* 登录页
* 注册页
* 忘记密码页

断点：

```css
@media (max-width: 800px) {
  ...
}
```

要求：

* 页面宽度不溢出。
* 表单卡片宽度使用百分比或 max-width。
* 大屏可左右分栏。
* 小屏必须单列。
* 按钮在手机端要容易点击。

### 主页

建议使用：

```css
@media (max-width: 1000px) {
  ...
}
```

要求：

* 大屏显示菜单和主体内容。
* 小屏隐藏或收缩菜单，只保留主要内容。
* 主体内容区域自适应。
* 图片、日历、卡片不能超出屏幕宽度。

### 用户列表

要求：

* 表格区域在小屏下不能破坏整体布局。
* 可给表格容器设置横向滚动，避免页面整体溢出。
* 分页组件在小屏下居中或换行展示。

## 八、验收演示流程

实现完成后，确保可以按以下流程演示：

1. 打开登录页面。
2. 点击注册用户，进入注册页面。
3. 注册页面填写错误邮箱，能提示错误。
4. 注册页面填写完整信息，能提示注册成功。
5. 返回登录页面。
6. 点击忘记密码，进入修改密码页面。
7. 输入不一致的新密码和确认密码，能提示错误。
8. 修改成功后返回登录页面。
9. 登录页输入用户名和密码，点击登录进入主页。
10. 主页大屏展示顶部栏、左侧菜单、欢迎信息、图片和主体内容。
11. 调整浏览器宽度，展示小屏响应式效果。
12. 点击蓝色风格和黄色风格，主题能切换。
13. 点击用户管理，进入用户列表。
14. 用户列表使用 el-table 展示静态数据。
15. 切换分页，页面数据能变化。
16. 点击信息页面，能展示信息内容。
17. 所有页面按钮、链接都能正常点击或给出合理提示。

完成后逐项检查：

* 登录、注册、忘记密码、主页、信息页、用户列表页都存在。
* 页面之间能跳转。
* 表单校验能触发。
* 注册页邮箱和出生年月验证存在。
* 修改密码页确认密码一致性验证存在。
* 用户名和密码非空验证存在。
* 主页能显示菜单、欢迎信息、图片和主体内容。
* 小屏下布局不溢出。
* 主题切换功能存在。
* 用户列表使用 el-table。
* 用户列表有分页组件。
* 数据来自前端静态模拟数组。
* 没有后端接口请求。
* 没有数据库代码。
* 没有 SpringBoot / MyBatis 内容。
* CSS 没有大量行内样式。
* 代码命名清楚。
* 组件和页面文件结构清楚。
* 验收时能完整演示网页效果。

## 十、实验报告可说明的实现点

代码完成后，项目应支持实验报告中说明以下内容：

1. 实验一只实现静态前端系统原型。
2. 页面包括登录、注册、忘记密码、主页、信息页和用户列表页。
3. 使用 Vue 管理页面状态和交互。
4. 使用 ElementUI 完成表单、按钮、表格、分页、菜单等组件。
5. 使用 CSS media query 实现响应式布局。
6. 表单校验包括非空、邮箱、出生年月、确认密码一致性。
7. 用户列表数据使用静态数组模拟。
8. 主题切换通过 CSS class 或主题变量实现。
9. 后端、数据库、真实数据读取将在后续实验二和实验三中实现，本实验不包含这些内容。
